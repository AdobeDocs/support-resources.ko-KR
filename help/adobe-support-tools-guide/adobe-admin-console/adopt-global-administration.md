# 글로벌 관리 채택

Enterprise에 적용됩니다.

Global Admin Console 사용의 이점에 대해 알아보고 액세스 방법을 알아보십시오. [자세히 알아보기](#request-access-to-the-global-admin-console)

Adobe Admin Console에서 지원 사례를 만들어 Global Admin Console 액세스를 요청합니다.
[Global Admin Console](https://adminconsole.adobe.com/support)에 로그인합니다.

## 소개

[Global Admin Console](https://adminconsole.adobe.com/support)은(는) 관리 콘솔이 많거나 기본 Admin Console을 여러 콘솔로 분리하려는 복잡한 조직 구조를 가진 고객을 위한 것입니다. 예를 들어, 다국적 기업, 교육 컨소시엄, 대규모 학군, 대형 정부 기관 등이 이에 해당한다. 기존 Admin Console을 조직 차트와 같은 계층 구조로 중첩하여 분산된 기업 전체에 걸쳐 투명성을 제공합니다.

Global Admin Console은 기존 Admin Console의 기능을 대체하거나 변경하지 않습니다. 이 워크플로우는 맨 위에 루트 조직이 있고 해당 루트 조직 아래에 모든 콘솔이 중첩된 계층 구조로 작동합니다. Admin Console이 루트 콘솔로 작동하는 계층 구조를 구매하고 만드는 데 중점을 둔 선택 기준이 있습니다.

Global Admin Console의 기능과 이점에 대해 자세히 알아보려면 [Global Admin Console 백서](https://community.adobe.com/questions-624/new-white-paper-for-the-adobe-global-admin-console-678929#M35625)를 참조하세요.

[!VIDEO](https://youtu.be/FLBWR78wpok)

이 비디오의 콘텐츠 및 보이스오버는 현재 영어로만 제공됩니다.

## 주요 이점

다음은 [!DNL Global Admin Console]의 몇 가지 주요 장점입니다.

- Adobe 제품 사용 및 클라우드 스토리지의 관리를 부서, 부서 또는 현장 사무실에 위임함으로써 중앙 집중식 관리의 필요성을 줄입니다.
- 각 엔티티에 대한 조직을 생성하고 계층 구조로 관리합니다. 조직을 삭제할 수도 있습니다.
- 여러 조직에 리소스 및 제품 라이선스를 배포합니다.
- 관리자가 할당된 범위 내에서 사용자와 리소스만 볼 수 있도록 조직 간 가시성을 제한합니다.
- 조직의 디렉터리 구조에 맞게 셀프서비스 방식으로 Admin Console 조직 구조를 관리합니다.
- 특정 조직이나 프로젝트에 대한 Adobe 리소스를 관리할 지정된 관리자를 할당하여 관리 센터를 만듭니다.
- 조직 수준에서 정책을 정의하고 적용합니다.
- 제품 프로필 및 사용자 그룹을 만들고, 편집하고, 삭제합니다.
- 보고를 위해 또는 다른 프로세스에서 사용하기 위해 조직 정보를 내보냅니다.
- 조직에 대한 업데이트 및 조직 계층에 대한 변경 사항을 가져옵니다.
- 예정된 만료, 만료 또는 비활성 계약을 포함하여 하위 조직의 ETLA 계약 상태에 대한 알림을 받습니다. 자세한 내용은 [계약 만료](https://helpx.adobe.com/kr/enterprise/using/contract-expiry.html) 설명서를 참조하십시오.

## 자격 기준

Adobe은 조직이 다음 기준 중 하나 이상을 충족하는 경우 전역 관리를 권장합니다.

- 여러 Admin Console을 관리하며 중앙 집중식 가시성 및 제어가 필요합니다.
- Admin Console을 분리하여 독립적으로 관리하려는 경우
- 여러 계약을 관리할 수 있습니다.
- 하위 조직 전체에 ID 구성을 분산하거나 중앙 집중화해야 합니다.
- 구매한 리소스를 기업의 다른 부분에 할당하고 별도의 관리자를 할당하려고 합니다.
- 특정 Adobe 제품, 지역 또는 조직의 관리자가 있으며 겹치거나 간섭하지 않도록 하려는 경우

## Global Admin Console 액세스 요청

[Global Admin Console](https://adminconsole.adobe.com/support/)에 대한 액세스를 요청하려면 Adobe 담당자에게 문의하고 아래 단계를 따르십시오.

1. [Admin Console](https://adminconsole.adobe.com/support/)에서 **[!UICONTROL 지원]** > **[!UICONTROL 지원 요약]**&#x200B;으로 이동합니다.

   >[!NOTE]
   >
   > 글로벌 관리자 계층에서 루트 조직 역할을 하는 기본 조직을 통해 지원 사례를 제출합니다.

1. **[!UICONTROL 사례 만들기]**&#x200B;를 선택하여 **[!UICONTROL 사례 만들기]** 창을 엽니다.

1. 양식에서 다음 세부 사항을 지정합니다.

   - 문제의 우선 순위: P4 - 사소
   - 문제의 영향: Medium
   - 문제 요약: &lt;조직 이름>에서 Global Admin Console 액세스를 요청하고 있습니다.

1. **[!UICONTROL 직면한 문제의 특성 설명]**&#x200B;에서 조직이 Global Admin Console 액세스를 요청한다고 명시하고 다음 세부 정보를 포함하십시오.

   - Admin Console의 **이름 및 콘솔 ID**(URL에서 &quot;@AdobeOrg&quot; 앞에 있는 일련의 숫자 및 문자로 표시됨)을(를) 사용하여 *루트(상위)* 조직으로 지정하거나 새 콘솔을 요청하고 원하는 이름을 지정하십시오.
- 경우에 따라 새 콘솔이 기본 옵션입니다. Adobe 담당자가 안내할 수 있습니다. 나중에 변경할 수 없으므로 *루트* 조직을 신중하게 선택하십시오.
   - 최소 한 명 이상의 현재 시스템 관리자를 전역 관리자 역할에 할당합니다. 나중에 관리자를 추가할 수 있습니다.
   - Adobe 담당자 또는 Customer Success Manager의 이름 및 이메일 주소.

   ![문제 설명](/help/adobe-support-tools-guide/assets/describe.png)

1. **[!UICONTROL 다음]**&#x200B;을(를) 선택하고 다음 정보를 제공하십시오.

   | 작성할 세부 정보 | 설명 |
   | ------------------ | ----------- |
   | **알릴 사람 나열** | Adobe 고객 지원 센터에서 사례 업데이트를 받으려면 최대 10개의 이메일 주소를 추가하십시오. |
   | **표준 시간대** | 호출이 필요한 경우 Adobe 고객 지원 센터에서 시간대를 사용합니다. 이 필드는 시스템 설정에 따라 자동으로 채워집니다. |
   | **작업 시간** | Adobe 고객 지원 센터는 통화가 필요한 경우 근무 시간과 정상 업무 시간을 고려합니다. 시간 슬라이더를 사용하여 가장 적합한 연락 시간을 지정하십시오. |
   | **전화 번호** | Adobe 고객 지원 담당자가 문제 해결 또는 설명을 위해 연락할 수 있도록 연락처 번호를 제공하십시오. |

1. **[!UICONTROL 제출]**&#x200B;을 선택합니다.
