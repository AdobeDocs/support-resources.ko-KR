# Global Admin Console에서 조직 정책 업데이트

**적용 대상:** Enterprise

글로벌 관리자가 Global Admin Console에서 조직 및 해당 하위 항목에 대한 정책을 설정하고 수정하는 방법에 대해 알아봅니다.

>[!NOTE]
>
>[Global Admin Console](https://helpx.adobe.com/kr/enterprise/global-admin-console/adopt-global-administration.html)에서 계층에서 조직을 선택하고 **정책** 탭으로 이동하여 정책을 허용하거나 허용하지 않거나 잠급니다.
>
> [Global Admin Console에 로그인](https://global-admin-console.adobe.com/)

정책은 조직과 연결되어 해당 조직에서 수행할 수 있는 작업을 제한합니다. 정책 값이 설정되면 해당 시점부터 작업을 제한하거나 활성화합니다.
예를 들어 **클레임 도메인** 정책이 *허용되지 않음*(으)로 설정된 경우 추가 도메인을 청구할 수 없지만 정책 값을 설정하기 전에 청구된 도메인은 영향을 받지 않습니다.

## 정책 구성

조직의 정책을 수정하려면 다음을 수행합니다.

1. Global Admin Console에서 [편집할 조직을 선택](https://helpx.adobe.com/kr/enterprise/global-admin-console/overview.html)한 다음 **[!UICONTROL 정책]** 탭으로 이동합니다.
1. 관련 정책에 대한 토글을 선택하여 허용하거나 허용하지 않습니다. [선택한 조직](https://helpx.adobe.com/kr/enterprise/global-admin-console/overview.html)의 전역 관리자 또는 상위 조직을 제외한 다른 사람이 정책을 변경하거나 잠금 해제할 수 없도록 정책을 잠글 수도 있습니다.
1. 정책을 잠그려면 **[!UICONTROL 잠금]** ![잠금](./assets/lock.png) 아이콘을 선택하십시오. 잠금 상태에서 마우스를 가져가면 선택한 조직의 이름이 표시됩니다. [정책 잠금](#policy-locks)에 대해 자세히 알아보세요.
1. 조직 편집을 마친 후 **[!UICONTROL 보류 중인 변경 내용 검토]**&#x200B;를 선택합니다. 검토 후 **[!UICONTROL 변경 내용 제출]**&#x200B;을(를) 선택하여 [실행](https://helpx.adobe.com/kr/enterprise/global-admin-console/execute-jobs.html)합니다.

## 정책 잠금 {#policy-locks}

정책이 잠겨 있으면 정책의 잠금이 해제될 때까지 해당 값을 변경할 수 없습니다. Global Admin Console은 조직 선택기에서 [선택한 조직](https://helpx.adobe.com/kr/enterprise/global-admin-console/overview.html)을(를) 정책이 잠긴 조직으로 기억합니다. 선택한 조직 또는 트리에서 상위 조직의 전역 관리자는 정책의 잠금을 해제할 수 있는 권한이 있습니다. 해당 조직보다 범위가 낮은 전역 관리자에게는 정책 값을 잠금 해제하고 변경할 수 있는 권한이 없습니다.

잠긴 환경을 생성하려면 하위 조직에서 원하는 정책 값을 설정한 다음 잠급니다. 이러한 하위 조직의 전역 관리자는 정책 값을 편집할 수 없습니다.

### 예: 잠긴 환경

*Acme Division*&#x200B;의 전역 관리자인 Elissa가 하위 조직 *마케팅* 및 *엔지니어링*&#x200B;을(를) 만든 다음 Robert를 *마케팅*&#x200B;의 전역 관리자로, Sarah를 *엔지니어링*&#x200B;의 전역 관리자로 추가합니다. 그런 다음 여러 정책을 *허용되지 않음* 및 *잠금*(으)로 설정합니다. Elissa가 *Acme Division*&#x200B;을(를) 선택한 조직으로 선택하면 나중에 정책 값을 잠금 해제하고 변경할 수 있지만 Robert와 Sarah는 조직 *Acme Division*&#x200B;에 의해 정책이 잠겨 있기 때문에 자신이 전역 관리자인 조직의 정책을 잠금 해제할 수 없습니다.

## 정책 세부 정보

### 조직 관리

| 정책 이름 | 설명 |
| --- | --- |
| **하위 조직 만들기** | 글로벌 관리자가 하위 조직을 만들 수 있도록 허용합니다. *off*&#x200B;이면 하위 조직을 만들 수 없습니다. |
| **조직 이름 바꾸기** | 허용되는 경우 글로벌 또는 시스템 관리자가 조직의 이름을 변경할 수 있습니다. 조직의 국가/지역 변경도 제어합니다. 상위 조직의 이름이 바뀌거나 해당 조직 또는 해당 조직의 상위 요소가 다시 지정되는 경우 이 정책 설정과 별도로 조직의 경로 이름을 변경할 수도 있습니다. |
| **조직 삭제** | 전역 관리자가 하위 조직을 삭제할 수 있도록 허용합니다. 사용자 자산을 삭제할 위험이 있어 엔터프라이즈 스토리지를 사용하는 조직이 활성화될 때 더욱 중요합니다. |

### 관리자 관리

| 정책 이름 | 설명 |
| --- | --- |
| **관리자 추가 또는 삭제** | 전역 관리자가 조직에 새 관리자를 추가할 수 있습니다. *off*&#x200B;인 경우 새 관리자를 추가할 수 없습니다. |
| **하위 조직을 만들 때 상위에서 시스템 관리자 상속** | 전역 관리자가 새 하위 조직을 만들면 상위의 시스템 관리자는 자동으로 새 조직의 시스템 관리자가 됩니다. 이 정책은 기본적으로 *off*&#x200B;입니다. |
| **관리자 관리** | 글로벌 관리자가 관리자 권한을 변경 또는 제거/편집할 수 있습니다. |

### 사용자 관리

| 정책 이름 | 설명 |
| --- | --- |
| **상위 조직에서 관리하는 디렉터리에서 사용자 상속** | 새 하위 조직을 만들기 전에 이 정책을 *on* 전환하고 활성화해야 합니다. 하위 조직이 생성되면 상위 조직의 사용자는 하위 조직의 사용자로 사용할 수 있게 됩니다. 즉, 이 정책은 GAC 내에서 새 하위 항목이 생성될 때 상위 항목과 하위 항목 간에 트러스트 관계를 자동으로 설정합니다. 기존 조직의 경우 GAC에 추가되기 전의 모든 트러스트 관계는 GAC로 가져온 후에도 유지됩니다. 만일 신뢰관계가 제자리에 있지 않았다면 통상의 신뢰요청절차를 따라야 한다. 이 정책이 성공하려면 새 조직을 만드는 전역 관리자도 요구한 도메인이 있는 상위 조직의 시스템 관리자여야 합니다. 그렇지 않으면 도메인 트러스트 관계가 새로 만든 조직에 상속되지 않습니다. |
| **Adobe ID 사용자 추가** | 설정된 경우 조직에서 Admin Console, UMAPI(사용자 관리 API) 또는 동기화 메커니즘을 통해 Adobe ID 유형 사용자를 추가할 수 없습니다. |
| **사용자 그룹 관리** | 허용되는 경우 글로벌, 시스템 및 사용자 그룹 관리자는 사용자 그룹을 생성, 편집 및 삭제할 수 있습니다. |

### 디렉터리 및 도메인 적용

| 정책 이름 | 설명 |
| --- | --- |
| **클레임 도메인** | 설정하면 시스템 관리자가 Admin Console에서 도메인을 요청할 수 있습니다. |
| **ID 구성 변경** | 설정하면 시스템 관리자가 Admin Console에서 사용자 ID 구성 설정을 변경할 수 있습니다. |

### 제품 할당

| 정책 이름 | 설명 |
| --- | --- |
| **제품 관리** | 글로벌 관리자가 제품을 추가 또는 제거하고 제품 리소스 부여를 변경할 수 있도록 허용합니다. |

### 자산 공유

| 정책 이름 | 설명 |
| --- | --- |
| **시스템 또는 저장소 관리자가 자산 공유 설정을 변경할 수 있습니다** | 스토리지 및 시스템 관리자가 보안 연락처, 암호 정책 및 스토리지 정책을 포함한 자산 공유 설정을 변경할 수 있습니다. |
| **조직을 만들 때 상위에서 공유 정책을 상속합니다** | 허용되는 경우 하위 조직이 생성될 때 자산 공유 설정이 상위에서 상속됩니다. 자산 공유 설정에는 보안 연락처, 암호 정책 및 저장소 정책이 포함됩니다. 이는 작성 시 새로 생성된 조직에만 적용됩니다. 이 값은 상위 항목에 설정되며 해당 상위 항목 아래의 하위 조직 만들기에 영향을 줍니다. |
