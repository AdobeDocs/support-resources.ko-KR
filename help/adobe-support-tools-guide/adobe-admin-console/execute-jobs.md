# 보류 중인 작업 실행

이 기능은 [[!DNL Global Admin Console]](https://global-admin-console.adobe.com/)을(를) 사용하는 Enterprise 조직에 적용됩니다.

- [[!DNL Global Admin Console]](https://global-admin-console.adobe.com/)의 변경 내용이 두 단계로 완료되었습니다.

   1. **단계 편집**: 조직을 변경하거나 제품을 할당합니다.
   2. **실행 단계**: 보류 중인 변경 내용을 검토하고 실행하여 적용하십시오.

- [[!DNL Global Admin Console]](https://helpx.adobe.com/enterprise/global-admin-console/adopt-global-administration.html)의 모든 변경 내용이 구현되고 적용되도록 하려면 **[!UICONTROL 작업 실행]** 탭을 선택하고 보류 중인 변경 내용 실행을 계속하십시오.

  [[!DNL Global Admin Console]](https://global-admin-console.adobe.com/)에 로그인합니다.

## 지속성 및 변경 사항 가시성

### 지속성 변경

- 보류 중인 변경 내용을 손실하지 않고 로그아웃했다가 나중에 다시 돌아갈 수 있습니다.
- 실행되지 않은 변경 사항:
   - 30일 후 삭제됩니다.
   - 브라우저 탭이나 창이 닫혀 있는 경우와 같이 세션이 종료되면 지워집니다.

&#x200B;> [!NOTE]
>
> 중요한 변경 사항을 신속하게 실행하여 성공적으로 적용되었는지 확인합니다.

### 여러 관리자 및 충돌

- 동일한 조직에서 작업 중인 두 명의 관리자:
   - 서로의 실행되지 않은 변경 사항은 볼 수 없습니다.
   - 다음 이후에만 변경 사항 보기:
      - 실행 및
      - 디스플레이를 새로 고치거나 다시 로그인하는 중입니다.
- 실행되지 않은 변경 사항은 이미 실행된 변경 사항과 충돌할 수 있습니다.

### 충돌 처리

충돌이 보고됨:
- 실행 시 또는
- 데이터를 새로 고친 경우(예: 로그아웃한 후 다시 로그인한 후).

## 보류 중인 변경 내용 실행

전역 관리자는 **[!UICONTROL 작업 실행]** 탭에서 보류 중인 변경 내용을 검토하고 실행할 수 있습니다.

### 변경 사항 실행 단계

1. [!DNL Global Admin Console]에 로그인합니다.
2. 왼쪽 탐색 패널에서 **[!UICONTROL 작업 실행]** 탭을 선택합니다.
3. 보류 중인 변경 내용을 검토합니다.
4. **[!UICONTROL 변경 내용 제출]**&#x200B;을 선택하여 실행하십시오.

작업을 제출한 후:

- 작업이 실행 큐에 들어갑니다.
- 작업이 실행되는 동안 상태는 **[!UICONTROL 보류 중]**&#x200B;입니다.
- Adobe에서는 예측 가능성과 보다 쉬운 문제 해결을 위해 한 번에 하나의 작업만 실행하는 것이 좋습니다.

&#x200B;> [!IMPORTANT]
>
> 실행 중에 오류가 발생하면 성공적으로 적용되지 않은 모든 변경 사항을 다시 입력하고 다시 제출해야 합니다.

### 장기 실행 할당

제품 할당이 12시간 이상 걸리는 경우:
- 작업이 **[!UICONTROL 실패]**(으)로 표시됩니다.
- 해당 작업에서 보류 중인 모든 후속 작업은 실행되지 않습니다.

![보류 중인 작업](assets/pending-jobs.png)

## 실행 중인 작업 취소

**[!UICONTROL 작업 실행]** 탭에서 현재 실행 중인 작업을 취소할 수 있습니다.

### 실행 중인 작업을 취소하는 단계

1. [!DNL Global Admin Console]에 로그인합니다.
2. **[!UICONTROL 작업 실행]**&#x200B;을 선택합니다.
3. **[!UICONTROL 실행 중인 작업 취소]**&#x200B;를 선택합니다.

### 취소 동작

1. 작업은 현재 단계가 끝날 때 중지됩니다.
2. 그 일은 한 걸음 한 걸음 내딛을 때 멈추지 않는다.
3. 일부 단계는 완료하는 데 몇 분 또는 몇 시간이 걸릴 수 있습니다.
4. 이 시간 동안 작업은 **[!UICONTROL 취소]** 상태로 유지될 수 있습니다.

&#x200B;> [!NOTE]
>
> 작업이 중지되면 현재 단계가 완료되면 취소가 상당히 지연될 수 있다는 점을 고려하여 취소를 계획합니다.

## 작업 내역 보기

- 지난 30일 동안 실행된 작업을 보려면 다음 작업을 수행하십시오.

   1. [!DNL Global Admin Console]에 로그인합니다.
   2. **[!UICONTROL 작업 실행]**&#x200B;을 선택합니다.
   3. 페이지 하단으로 스크롤합니다.
   4. **[!UICONTROL 최근 작업]**&#x200B;을 선택하십시오.

- 최근 작업 표시:
   - **작업 명령**&#x200B;을(를) 제출했습니다.
   - 실행과 연결된 **오류** 및 **경고**.

&#x200B;> [!NOTE]
>
> 관련 개체의 이름을 바꾸거나 삭제하면 **명령이 작업 기록에 표시되는 방식에 영향을 주지 않습니다**. 기록은 제출 시점의 상태를 반영합니다.
