# Adobe Admin Console의 접근성

엔터프라이즈 및 팀에 적용됩니다.

전체 조직에서 Adobe 권한을 관리할 중앙 위치를 제공하는 [!DNL Adobe Admin Console]의 접근성 지원에 대해 알아봅니다.

## 운영 체제 접근성 지원

널리 사용되는 여러 운영 체제의 접근성 기능을 [!DNL Admin Console]에서 사용할 수 있습니다. 이러한 접근성 기능에는 고대비 및 기타 디스플레이 설정이 포함됩니다. 그러나 [!DNL Admin Console]의 현재 사용자 인터페이스로 인해 몇 가지 제한 사항이 있을 수 있습니다.

## 키보드 지원

[!DNL Admin Console]의 대화형 콘텐츠 대부분은 키보드만 사용하여 탐색하고 작동할 수 있습니다. 예를들어 탐색 및 컨트롤에 **[!UICONTROL Tab]** 또는 **[!UICONTROL Shift]** + **[!UICONTROL Tab]**&#x200B;을 사용할 수 있습니다. 올바른 링크 또는 단추로 이동한 후 **[!UICONTROL Enter]** 또는 **[!UICONTROL Return]** 키를 눌러 선택합니다. [접근성 적합성 보고서](https://www.adobe.com/accessibility/compliance.html)에 설명된 대로 일부 예외가 있을 수 있습니다(_영어만_).

## 보조 기술 지원

[!DNL Admin Console]은(는) 보조 기술(화면 판독기 및 화면 돋보기 포함)에 대한 일부 지원을 제공합니다. 자세한 내용은 [접근성 적합성 보고서](https://www.adobe.com/accessibility/compliance.html)를 참조하십시오.

## Admin Console에서 지원되는 접근성 기능은 무엇입니까?

[[!DNL Admin Console]](https://adminconsole.adobe.com/)은(는) 전체 조직에서 Adobe 권한을 관리할 중앙 위치를 제공합니다. 보조 기술 지원, 키보드 지원 및 운영 체제 접근성 지원(일부 예외)을 제공합니다.

## 피드백이 있습니까?

[!DNL Admin Console] 팀은 접근성 및 사용자 환경을 개선하기 위해 지속적으로 노력하고 있습니다. [엔터프라이즈 및 팀 커뮤니티](https://community.adobe.com/enterprise-teams-622)에서 접근성 기능에 대한 피드백을 공유할 수 있습니다.
